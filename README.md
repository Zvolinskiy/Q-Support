# Q-Support — Социальное приложение для экстренной помощи

**Q-Support** — это социальное приложение, созданное для оказания оперативной помощи в экстренных ситуациях. Цель проекта — спасать жизни через доступность критически важной информации. Приложение работает в **полном оффлайн-режиме**, что гарантирует стабильный доступ к данным даже при отсутствии связи.

---

## Описание функциональных модулей

### 1. База экстренных алгоритмов
В приложении реализован структурированный справочник по оказанию первой доврачебной помощи и действиям в ЧС. 
* **Категоризация:** Инструкции разделены на логические блоки (Медицина, Безопасность, Право). Например, в категории **«Отравления»** доступен пошаговый протокол из 1–5 строк.
* **Принцип «Золотого часа»:** Контент оптимизирован для быстрого чтения в стрессовой ситуации, когда на счету каждая секунда.
* **Хранение данных:** Все инструкции, переводы и системные тексты подгружаются из локального файла `/assets/data.json`.

### 2. Поиск и службы поддержки
Реализован быстрый поиск, работающий по внутренней базе данных приложения.
* **Службы поддержки:** База включает актуальные контакты экстренных ведомств: от скорой помощи и МЧС до юридических консультаций и линий противодействия коррупции.
* **Мгновенный доступ:** Алгоритм ищет по ключевым словам, позволяя найти номер службы или нужную инструкцию за доли секунды.

### 3. Модуль «Волонтеры» и Бот-помощник
В приложение добавлена кнопка «Волонтеры» для получения оперативной помощи. 

* **Текущая реализация (AI-Бот):** Сейчас в этом модуле работает бот, который помогает найти решение проблемы через текстовый запрос.
    * **Логика на весах слов:** Бот анализирует входящие слова и присваивает им баллы (веса). Система суммирует эти баллы: если тема «ДТП» набирает наибольший вес, бот выдает соответствующую инструкцию.
    * **Безопасность (Фильтр матов):** Внедрена система блокировки нецензурной лексики на любом языке. База для фильтрации находится в файле `/assets/filters_data.bin`.
    * **База знаний:** Бот содержит 15+ детальных инструкций, а также понимает приветствия, слова благодарности и системные вопросы.
* **Планы на будущее (Roadmap):** Создание платформы для связи с живыми людьми, возможность отправки фотографий с места событий, записи голосовых сообщений и прямых звонков.

### 4. Специальные возможности и настройки
* **Персонализация Интерфейса:** Скрытая функция в настройках, открывающая доступ к личному блокноту для фиксации данных «на лету» (ФИО сотрудников, названия лекарств, приметы) в экстренной ситуации.
* **Шрифт под зрение:** Добавлена настройка размера текста от **14sp до 28sp**, чтобы его было видно издалека или людям с плохим зрением.
* **Собственная озвучка (TalkBack):** Реализована индивидуальная система озвучки интерфейса, которая адаптирована под логику приложения и работает стабильнее стандартных экранных дикторов.
* **Удаление данных:** Одной кнопкой можно мгновенно стереть всю историю и зашифрованные заметки.

## Техническая архитектура

* **Модель:** Приложение базируется на **монолитной архитектуре** (основной код в `MainActivity.java`). Это обеспечивает бесшовную работу с данными и стабильность сессий без лишних затрат на переключения между экранами.
* **UI:** В `activity_main.xml` использованы динамические layout-контейнеры.
* **Диагностика:** Внедрен кастомный `CrashHandler` для выявления багов на этапе старта.
* **Стек:** Проект совмещает логику на **Java** и производительность **C++** (через JNI).

## Алгоритм шифрования заметок

Защита данных в разделе «Персонализация Интерфейса» реализована на уровне Native-слоя (C++):
* **Шифрование:** Данные + Ключ → XOR 0xAF.
* **Дешифровка:** Данные XOR 0xAF → - Ключ.
* **Реализация:** Вызывается через JNI для маскировки строк в оперативной памяти «на лету».

## Системные требования

* **ОС:** Android 10.0 (API 29) и выше.
* **Связь:** Не требуется (полный Offline).
* **Языки в приложении:** Русский, Казахский, Английский.
